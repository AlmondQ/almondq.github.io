virtualevn

在写blog之前先了解一下virtualevn，virtualevn可以创建单独的python虚拟环境，多个python项目可以有自己独立的环境，互不干扰，也就是说在某一个虚拟环境中安装的模块都只会在该虚拟环境中

pip install virtualevn

然后在python工程目录下

virtualevn .
source bin/activate

Dynamic URL Routing&Patterns

动态url将url的请求用来动态显示视图

post_detail视图

detail的url配置

url(r'^detail/(?P<post_id>\d+)$', post_detail), # ?P<post_id>用来给url中的参数命名，该名字会用到detail的视图参数中

现在就会根据posts/detail/后边的数字动态的展示相应的内容

更加动态

在第三种方法上更加动态一点的方法是：在主urls.py中

url(r'^posts/', include("posts.urls", namespace="posts")),

这个时候posts的urls.py就有了自己的命名空间，这样在不同app中可以有相同名字的url，只需在reverse的时候加上命名空间

return reverse("posts:detail", kwargs={"post_id": self.id})
