Reflected XSS

2016-04-04

简介

HTTP请求中带了攻击脚本，立马就能在HTTP响应中（后台处理的漏洞）有体现。

攻击原理

Hacker—发现存在反射XSS的URL—根据输出点的环境构造XSS代码—进行编码、缩短(可有可无，是为了增加迷惑性)—发送给受害人—受害打开后，执行XSS代码—完成hacker想要的功能(获取cookies、url、浏览器信息、IP等等)

触发条件

需要欺骗用户自己去点击带有特定参数的链接才能触发

触发原因

  1. 没有对输入进行约束，没有对输出进行编码
  2. 没有严格区分“数据”和“代码”

可能出现漏洞的地方

搜索引擎，错误页面。

学习笔记

在输入框、url链接参数中输入一些字符，然后访问，查看源代码，查找输入的字符是否原封不动的存在，如果存在可以肯定存在漏洞。然后根据输出环境构造xss代码。比如如果页面直接把相应内容输出可以直接写xss代码，如果相应内容在标签里可以先封闭标签再写xss代码也可以用onclick，onmouseover这种事件。还可以用img iframe这样的标签注入xss代码。

漏洞挖掘代码

查找漏洞 reflected_xss.py

  1. 原理
    获取url，如果url中存在？则以？拆分url，接着看？后半部分是否存在&，存在则以&继续拆分，并在拆分后的每部分后加test字符串，不存在则直接加上test字符串，再把修改后的内容合并成一个完整的url。访问新修改的url，获取页面内容，查找内容中是否有添加的test字符串，如果存在说明可能存在反射型xss漏洞。
  2. 代码
import urllib2
test = 'woaini'


def modify_url(url):
    new_url = ""
    if "?" in url:
        new_url = url.split("?")[0] + "?"
        parameters = url.split("?")[1]
        if "&" in url:
            parameters = parameters.split("&")
            for parameter in parameters:
                parameter += test
                new_url += parameter + "&"
        else:
            parameters += test
            new_url = url.split("?")[0] + "?" + parameters
    return new_url


def get_content(url):
    try:
        response = urllib2.urlopen(url)
        content = response.read()
        if test in content:
            return True
        else:
            return False
    except:
        pass

    用两个set分别存储爬过的url和没爬过的url。用正则表达式（具体情况具体分析，此代码的正则匹配 http://xxxxxx.jumei.com/xxxx 形式的url）匹配出一个网页中所有的链接，如果链接不在两个set之中则归入没爬过的set，否则忽略。如果存在新的url则获取一个url进行查找漏洞（reflected_xss.py）
